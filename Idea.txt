Project Idea (Quick Full Recap)
Project Name

Vision-to-Voice: Smart Classroom Assistant for the Visually Impaired

Core Goal

Convert whiteboard content (text, math, diagrams) into spoken explanations so visually impaired students can understand classroom teaching in real time.

The system looks at the board → understands it → explains it in audio.

High-Level Concept

A camera captures the whiteboard.
AI models analyze what is written or drawn.
The system generates a simplified explanation.
The explanation is converted into speech for the user.

Step-by-Step Explanation
1. Image Input Layer

The system first captures an image of the whiteboard.

It supports three possible input sources:

Raspberry Pi Mode

Camera connected to Raspberry Pi captures the board.

Camera → Raspberry Pi → send image to system

Pi acts as an edge capture device.

Laptop Mode

If Raspberry Pi is not available:

Laptop webcam → capture snapshot
Upload Mode (Testing)

During development or demo:

User uploads image → system processes it
2. Image Preprocessing (OpenCV)

The captured image is cleaned before analysis.

Typical operations:

Resize image

Convert to grayscale

Improve contrast

Reduce noise

Optional edge detection

Purpose:
Improve OCR accuracy and visual understanding.

3. OCR Layer (Text & Math Extraction)

OCR extracts written content from the board.

Example output:

x² + 5x + 6 = 0

Possible tools:

PaddleOCR

TrOCR

This step extracts:

Handwritten text

Mathematical expressions

Labels in diagrams

4. Vision-Language Model (NVIDIA API)

This is the main intelligence layer.

The image and extracted text are sent to a Vision-Language Model.

Example model:
NVIDIA NIM – LLaVA

The model understands:

diagrams

charts

written explanations

board context

Example output:

"The teacher is solving a quadratic equation on the board."

5. Explanation Engine (Your Logic Layer)

This layer transforms the raw AI output into student-friendly explanations.

Responsibilities:

Simplify language

Break explanations into steps

Adjust explanation length

Adapt explanation style

Example transformation:

Model output:

Quadratic equation x² + 5x + 6 = 0

Explanation engine output:

“The teacher is solving a quadratic equation. This equation can be factored into two brackets.”

This module is RL-ready if you later add behavior adaptation.

6. Text-to-Speech

The explanation text is converted into spoken audio.

Possible tools:

Coqui TTS

Mozilla TTS

Example:

Text → speech waveform
7. Audio Output

The generated speech is played for the user through:

Laptop speakers

Headphones

Phone audio (optional)

Example output:

“The teacher is solving a quadratic equation…”

Raspberry Pi Integration (Optional Edge Layer)

Raspberry Pi is used only for camera capture.

It does not run AI models.

Architecture:

Raspberry Pi Camera
        ↓
Pi captures image
        ↓
Image sent via HTTP
        ↓
Main AI system processes image

This makes the system:

✔ hardware-agnostic
✔ easy to test without Pi
✔ easy to demo with Pi

Dual Mode Operation

Your system works in two modes.

Mode 1 – Edge Mode
Camera → Raspberry Pi → AI pipeline

Used during hardware demo.

Mode 2 – Software Mode
Laptop webcam / image upload → AI pipeline

Used during development.
